# Terrain Generation Plan — hedgeFun

## Goal

Replace the flat, uniform ground with a fully procedural landscape generated per
playthrough from a seed. Each session gives a meaningfully different terrain that
influences where and how the player chooses to plant their hedge.

---

## Current state

- Ground is a fixed horizontal line at row 20 across all 200 columns
- Underground has uniform topsoil / subsoil / bedrock layers based on absolute row number
- Fertility and rock density are generated by simple sine-wave column noise
- Decorative boulders and aquifers exist as static overlay art, with no gameplay effect
- SoilMap does not know the terrain seed; everything is deterministic from position alone

---

## TerrainMap — core data generated per seed

`src/simulation/TerrainMap.ts` is the single source of truth for terrain data. All
other systems read from it. It is generated once in `GameScene.create()` and
persisted via `terrainSeed` in SaveData.

### Per-column arrays (length = GRID_CONFIG.cols)

| Field | Type | Range | Description |
|---|---|---|---|
| `groundRows` | `number[]` | 17–23 | Surface row for each column (smooth noise) |
| `biomes` | `SoilBiome[]` | 0–4 | Soil character of each column |
| `bedrockRows` | `number[]` | groundRow+19 to groundRow+31 | Depth at which bedrock starts |

### Structural features (sparse objects)

| Feature | Count | Description |
|---|---|---|
| `AquiferData[]` | 2–4 | Underground water bands at subsoil depth |
| `BoulderMass[]` | 20–35 | Large rocky masses that block root growth |
| `ClayLens[]` | 1–3 | Horizontal impermeable bands in subsoil |
| `springCols` | `Set<number>` | Columns where an aquifer breaks near surface |

### Soil biomes

```
Loam   — rich, dark brown. Balanced fertility, low rock. Most species do well.
Clay   — heavy, compact orange-brown. Holds water. High rock density.
Chalk  — pale grey-white. Alkaline. Sparse fertility, stony.
Peat   — very dark, acidic, wet. High fertility, soft. Willow loves it.
Sandy  — light tan, dry, fast draining. Low fertility, low rock.
```

Biomes are generated as contiguous horizontal zones (20–60 cols each) assigned
randomly from the seed. They affect:
- Underground visual appearance (different glyph colors per biome)
- SoilMap fertility and rock density modifiers
- Species growth rate modifier (see species sensitivity table)

## Visual changes

### Surface height variation

The ground row shifts ±3 rows across the map. Background rendering in
`GlyphAtlas` becomes **depth-relative** rather than absolute-row-based:

- Depth 0 = this column's ground row (ground layer glyphs)
- Depth 1–7 = topsoil
- Depth 8–20 = subsoil
- Depth 21+ = bedrock (or sooner if bedrockRow reached)
- Above ground rows 8–(groundRow-1) = vegetation zone (proportional canopy/shrub)
- Rows 0–7 = sky always

Slope transitions between adjacent columns get a `/` or `\` glyph.

### Biome surface hints

Grass glyphs at the ground row vary by biome:
- Loam: standard green `"` `'` `,`
- Clay: darker, sparser grass with occasional `_` flat patches
- Chalk: pale glyphs, occasional `·` white pebble
- Peat: dark mossy `%` `~` with near-black ground
- Sandy: sparse `'` `.` with tan color

### Underground features (overlays, placed once at startup)

- **Aquifers**: sinusoidal `~` `≈` bands in blue-grey, drawn from TerrainMap data
- **Boulder masses**: multi-cell rock formations using existing template art
- **Clay lenses**: horizontal `=` bands in orange-brown
- **Springs**: `~` at surface ground row where aquifer is near surface

## System changes

### GlyphAtlas (`src/rendering/GlyphAtlas.ts`)

Signatures change to accept `colGroundRow`:

```typescript
getBackgroundGlyph(col, row, colGroundRow): Glyph
getBackgroundColor(row, colGroundRow): string
```

Underground layer is determined by `depth = row - colGroundRow`, not absolute row.
Above-ground vegetation zone uses proportional layer assignment.
Biome glyph color sets added (5 variants for topsoil/subsoil).

### AsciiRenderer (`src/rendering/AsciiRenderer.ts`)

New method: `setGroundRows(rows: number[]): void`

`renderFullGrid` becomes column-aware:
- Reads `this.groundRows[col]` each cell (fast array lookup)
- Passes colGroundRow to GlyphAtlas functions
- Sky color only applied to rows 0–7 (unchanged)

### SoilMap (`src/simulation/SoilMap.ts`)

Constructor takes `TerrainMap` instead of fixed `groundRow`.
- `getSoilAt(col, row)`: depth-relative layer detection, biome modifier, boulder/clay lens/aquifer effects
- `getColumnQuality(col, rootDepth)`: uses per-column ground row from TerrainMap
- `getSoilLayer(row)` removed (replaced by depth-based logic)

### GameScene (`src/scenes/GameScene.ts`)

- `soilMap` becomes a class field (needed by decoration method)
- `terrainMap: TerrainMap` and `terrainSeed: number` added
- `private readonly groundRow` removed; replaced by `getColGroundRow(col)` helper
- Cursor A/D movement: `setCursor(col ± 1, terrainMap.getGroundRow(col ± 1))`
- Click-to-move: cursor snaps to terrain height at clicked column
- `tryPlant()`: roots plant at `terrainMap.getGroundRow(col)`
- `addGroundDecoration` → `addTerrainDecoration()` — fully rewritten, deterministic
  via terrain seed hash, replaces old `addBoulders` / `addAquifers` dead code
- Camera view modes use `terrainMap.getMaxGroundRow()` for bounds
- SaveData: `terrainSeed` persisted; old saves without it get flat terrain fallback

## Risk areas

- `AsciiRenderer.renderFullGrid` is the hot path (11,000 cells/frame). Moving
  bgColor computation inside the col loop is a minor cost — still negligible in JS.
- Cursor following terrain changes feel. Test on a real playthrough to tune height
  range (±3 rows may want adjusting).
- Old saves without `terrainSeed` use `TerrainMap.createFlat()` — flat ground,
  uniform loam — so existing progress is not broken by the upgrade.
- `PlantRenderer` already uses `plant.row` for positioning, so variable-height
  plants work automatically once `tryPlant` passes the correct ground row.
